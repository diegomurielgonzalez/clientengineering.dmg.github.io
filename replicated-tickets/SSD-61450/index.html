Left-Aligned Glia Bubble & Engagement Panel

Overrides Glia's native positioning to dock the interface to the bottom-left locally.

CSS Override Layout Responsive

Welcome to the Mock Site

The Glia bubble is anchored to the bottom left.

🔗 Jira Ticket: SSD-61450

Client Context

The client needs the Glia layout docked to the left on a specific testing site. Since they might not want this layout applied globally across all their domains, we cannot use the global 'Engagement Panel Location' setting. The solution must be fully responsive and overcome Glia's native 768px desktop flexbox breakpoint, all without stripping away the native inner padding or causing mobile viewport overflow.

Steps to Fix

  • Deploy the standard Glia integration script on the page.
  • Inject the following custom CSS ruleset strictly targeting the outermost wrappers: #salemove .sm-touch-draggable-lower-right (the bubble) and #salemove .sm-touch-draggable-placeholder (the open layout).
/* =========================================================
   REPLICATION-SPECIFIC CSS: LEFT-ALIGNED BUBBLE
   ========================================================= */
   
/* 
 * LEFT-ALIGN BOTH STATES:
 * - .sm-touch-draggable-lower-right .sm-visitor-app = bubble (closed)
 * - .sm-touch-draggable-placeholder .sm-visitor-app = panel (open)
 *
 * Uses Glia's own --horizontal-offset and --vertical-offset variables
 * but applies them to left/bottom instead of right/bottom.
 */
#salemove .sm-visitor-app-container .sm-visitor-app {
    position: fixed !important;
    left: var(--horizontal-offset) !important;
    right: auto !important;
    bottom: var(--vertical-offset) !important;
}

/* Also cover the omnibrowse container (CoBrowsing/screenshare UI) */
#salemove .omnibrowse .ob-visitor-container {
    left: var(--horizontal-offset) !important;
    right: auto !important;
    bottom: var(--vertical-offset) !important;
}

/* Mobile adjustments */
@media (max-width: 768px) {
    #salemove .sm-visitor-app-container .sm-visitor-app,
    #salemove .omnibrowse .ob-visitor-container {
        left: 20px !important;
        bottom: 20px !important;
    }
}
  • Apply position: fixed !important, left, and bottom coordinates to detach these wrappers from Glia's native right-aligned flexbox container. Do not target .sm-visitor-app to ensure native padding is preserved.
  • Include a mobile media query (max-width: 768px) to slightly tighten the left/bottom margins and apply a max-width: calc(100vw - 40px) strictly to the open layout wrapper so it does not bleed off smaller screens.
